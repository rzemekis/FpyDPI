# DPI Bypass — SOCKS5-прокси с GUI Теперь кроссплатформенный(Windows + Linux)

# Licensed under GPL-3 License//Защищен лицензией GPL-3

# Тесты проводились на cachyos с ядром linux 7.0.5-2-cachyos и на Win10

Пользовательский SOCKS5-прокси, который обходит DPI-блокировки путём
фрагментации TCP, разбивки TLS-записей и рандомизации регистра HTTP-заголовка `Host`.
Предназначен для разблокировки YouTube и аналогичных сервисов. Рут не нужен.

## Установка при использовании source-кода

```bash
python3 -m venv .venv
# для линукса
source .venv/bin/activate
# для винды
.venv\Scripts\activate
#общая
pip install -r requirements.txt
```

## Запуск source-кода

```bash
python main.py
```

Нажать **Start**. Логи отображаются на вкладке Dashboard.

## Настройка браузера

Укажи в браузере SOCKS5-прокси на `127.0.0.1:8881` (или другой порт из Settings).

### Firefox

1. `about:preferences` → **Параметры сети** → **Настроить…**
2. **Ручная настройка прокси**
3. **SOCKS-узел** = `127.0.0.1`, **Порт** = `8881`, выбрать **SOCKS v5**
4. Поставить галку **Использовать DNS через SOCKS v5**
5. Сохранить

### Chromium / Chrome

```bash
google-chrome --proxy-server="socks5://127.0.0.1:8881"
```

### Проверка через curl

```bash
curl --socks5-hostname 127.0.0.1:8881 https://www.youtube.com -I
```

## Список доменов (fragmentthis.txt)

Файл `fragmentthis.txt` должен лежать **рядом с бинарником** (или рядом с `main.py`
при запуске из исходников). Один домен на строку. Поддомены совпадают автоматически:
`youtube.com` захватит `www.youtube.com`, `music.youtube.com` и т.д.
Строки начинающиеся с `#` игнорируются. Если файл пустой или отсутствует —
обход применяется ко **всем** подключениям.

```
youtube.com
youtu.be
googlevideo.com
```

Кнопка **↺ Reload routing** в Dashboard перечитывает файл без перезапуска прокси.

## Как работает обход

1. Браузер открывает TCP через SOCKS5 к нужному хосту.
2. Прокси определяет тип первого чанка: TLS `ClientHello` или HTTP-запрос.
3. **TLS**: при включённом `TLS record split` разбивает один TLS-рекорд на два
   валидных — разрез делается внутри SNI-имени. DPI, ищущий имя хоста в одном
   рекорде, не видит его целиком. Опционально: OOB-байт между половинами,
   fake-TLS-рекорд перед настоящим ClientHello.
4. **HTTP**: реверс регистра заголовка `Host` (`hOsT: youtube.com`) + фрагментация
   по 1–10 байт. Простые DPI-сигнатуры не распознают заголовок.
5. Дальнейший трафик проксируется без изменений.

Техника аналогична GoodbyeDPI / zapret / byedpi, но реализована как чистый
SOCKS5-прокси в пространстве пользователя — никаких NFQUEUE, raw-сокетов и
kernel-модулей.

## Блокировка QUIC

Браузеры предпочитают HTTP/3 (QUIC/UDP) там, где он доступен. SOCKS5-прокси
UDP не перехватывает. Включи **Block QUIC (UDP 443)** в Settings - прокси откатит
браузер на TCP/TLS и будет проходить через прокси.

## Настройки

| Параметр | Описание |
|---|---|
| Proxy port | Порт SOCKS5-прокси (по умолчанию 8881) |
| Fragment size | Размер чанка при TCP-фрагментации (1–10 байт) |
| Split TLS inside SNI | Разрезать ClientHello прямо внутри SNI-имени |
| TLS record split | Разбить на два валидных TLS-рекорда (рекомендуется) |
| Inject OOB byte | OOB-байт между половинами (экспериментально) |
| Fake TLS record | Фейковый TLS-рекорд перед настоящим (обычно ломает TLS) |
| Host case toggle | Рандомизировать регистр заголовка Host в HTTP |
| Block QUIC | Заблокировать UDP/443 через iptables |

## Сборка бинарника

```bash
#для линукса
source .venv/bin/activate
#для винды
.venv\Scripts\activate
#общая
pip install pyinstaller
pyinstaller dpibypass.spec --clean # в зависимости от ос соберет executable для линукса или .exe для винды
```

Бинарник появится в `dist/dpibypass`. Скопируй рядом `fragmentthis.txt` со
своим списком доменов.

## Файлы

- `main.py` — CTK GUI, трей, настройки
- `proxy.py` — SOCKS5-сервер + движок обхода DPI
- `quic_blocker.py` — блокировка UDP/443
- `fragmentthis.txt` — список доменов для обхода (один домен в строке)
- `dpibypass.spec` — конфиг PyInstaller
- `requirements.txt` — Python-зависимости
